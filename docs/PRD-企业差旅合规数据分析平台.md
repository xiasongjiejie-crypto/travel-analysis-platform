## 2. 项目背景

企业差旅费用是可控成本中占比高、发生频次高、审核难度大的一类支出。传统的人工抽查审核方式存在以下痛点：

- **覆盖率低**：人工只能抽查少量单据，绝大多数违规无法被发现。
- **模式单一**：只能发现"超标"这类显性违规，对拆分报销、司乘合谋、时空矛盾等隐蔽违规无能为力。
- **滞后性强**：事后审计，无法在报销发生时实时拦截。
- **缺乏画像**：无法沉淀员工/商户/司机的历史行为，难以识别系统性风险。

本项目旨在构建一个**数据驱动的差旅合规分析平台**，通过规则引擎 + 异常检测 + 关联分析，实现差旅费用的自动化合规审查、风险评分与预警。

---

## 3. 项目目标

### 3.1 业务目标
- 提升违规发现率，覆盖机票、酒店、用车、餐饮、报销全链路。
- 从"事后抽查"升级为"全量筛查 + 实时预警"。
- 沉淀员工、商户、司机的行为画像，识别系统性/团伙性违规。

### 3.2 产品目标
- 提供可配置的**规则引擎**，业务人员可自定义合规规则与阈值。
- 提供**风险评分与分级**，帮助审核人员聚焦高风险单据。
- 提供**告警与复核工作流**，形成"发现—复核—处置"闭环。
- 提供**合规看板与报表**，支撑管理决策。

### 3.3 衡量指标（成功标准）
- 违规检出率相较人工抽查提升 ≥ 3 倍。
- 硬规则告警准确率 ≥ 95%，软规则复核命中率 ≥ 30%。
- 单批次全量数据分析耗时可接受（满足 T+1 或近实时）。

## 4. 名词术语

| 术语 | 定义 |
|------|------|
| 差标 | 差旅标准，按员工职级、城市分级设定的费用上限 |
| 硬规则 | 确定性违规规则，命中即告警（如超标、无审批、重复发票） |
| 软规则 | 可疑规则，命中后打分并进入人工复核 |
| 司乘亲密度 | 一段时间内乘客搭乘特定车牌/司机的集中程度指标 |
| HHI | 赫芬达尔指数，衡量集中度 |
| 风险分 | 综合多规则命中情况计算出的单据/员工风险评分 |
| 时空矛盾 | 同一员工在时间重叠的两笔消费地理距离超过可移动范围 |

## 7. 功能需求详述

### 7.1 数据接入与清洗（F-DATA）

- **F-DATA-01 多源数据接入**：支持机票、酒店、企业滴滴用车、餐饮、发票、出差申请单、员工差标等数据的接入（CSV / 数据库 / API）。
- **F-DATA-02 数据标准化**：统一字段格式、时间格式、金额单位、地理坐标。
- **F-DATA-03 数据关联**：以 `employee_id`、`travel_apply_id`、`invoice_no` 等为主键关联多源数据。
- **F-DATA-04 数据质量校验**：缺失值、异常值、重复记录检测与标记。

### 7.2 规则引擎（F-RULE）

- **F-RULE-01 规则配置**：管理员可配置规则的启用/停用、阈值、适用范围。
- **F-RULE-02 三层规则体系**：硬规则（命中即告警）、软规则（打分复核）、模型层（异常检测）。
- **F-RULE-03 白名单机制**：支持对特定场景（通勤班车、驻场用车等）设置白名单。
- **F-RULE-04 规则命中记录**：记录每条单据命中的规则与依据，可追溯。

> 具体规则清单见 **附录 A：合规规则库**。

### 7.3 司乘亲密度分析（F-AFFINITY）★核心特色

- **F-AFFINITY-01 集中度计算**：按员工在时间窗口内，计算最高频车牌的行程占比（重复率 R）。
- **F-AFFINITY-02 HHI 指数**：计算赫芬达尔指数，识别绑定 2~3 个固定司机的情况。
- **F-AFFINITY-03 概率显著性检验**：基于城市司机密度计算随机期望重复次数，判断实际重复是否显著异常，自动适配城市规模，降低误报。
- **F-AFFINITY-04 司机侧反查**：统计某司机的乘客构成，识别"司机大部分单来自同一员工"。
- **F-AFFINITY-05 图/团伙分析**：构建"员工—车牌"二部图，识别强绑定边与共享车牌的团伙。
- **F-AFFINITY-06 最小样本门槛**：行程数低于阈值不判定，避免小样本误报。

### 7.4 风险评分与分级（F-SCORE）

- **F-SCORE-01 单据风险分**：聚合单据命中的规则权重，计算风险分。
- **F-SCORE-02 员工风险画像**：聚合员工历史行为与命中记录，形成风险画像。
- **F-SCORE-03 风险分级**：按分值划分低/中/高风险等级。
- **F-SCORE-04 风险升级**：多信号叠加时（如司乘亲密度高 + 非工作时段 + 空驶）自动升级风险等级。

### 7.5 告警与复核工作流（F-FLOW）

- **F-FLOW-01 告警生成**：硬规则命中直接告警，软规则/模型命中进入复核队列。
- **F-FLOW-02 复核处置**：审核人员可对告警进行确认违规/误报/需补充材料等处置。
- **F-FLOW-03 处置反馈**：处置结果回流用于优化规则阈值、降低误报。
- **F-FLOW-04 通知**：高风险告警支持通知相关审核/审计人员。

### 7.6 合规看板与报表（F-DASH）

- **F-DASH-01 合规总览**：违规总量、金额、检出率、趋势。
- **F-DASH-02 维度分析**：按部门、城市、费用类型、员工职级下钻。
- **F-DASH-03 高风险榜单**：高风险员工/商户/司机排行。
- **F-DASH-04 报表导出**：支持审计报告导出。

## 8. 违规场景总览（业务需求来源）

平台需覆盖以下五大环节 + 系统性违规，详细规则见附录 A。

### 8.1 机票 / 交通
- 提前预订不足 / 临时抢票
- 超舱等级乘坐
- 退改签异常（套现）
- 绕行 / 非最优路线

### 8.2 酒店 / 住宿
- 超标准入住
- 住宿天数与行程不匹配
- 同城 / 本地住宿
- 拼房未共享 / 重复报销

### 8.3 用车 / 打车
- 非工作时间 / 非工作日打车
- 行程轨迹异常（起终点为居住地/娱乐场所）
- 高频 / 拆单打车
- 有公共交通仍打车
- **司乘亲密度异常（核心）**

### 8.4 餐饮 / 招待
- 超标准餐费
- 招待费与人数/事由不符
- 出差期间双重餐补
- 非出差期 / 非出差地餐饮

### 8.5 报销 / 发票（跨环节核心）
- 重复报销
- 拆分报销规避审批
- 发票真伪 / 抬头异常 / 连号
- 报销与出差申请不匹配
- 消费时间/地点冲突（时空矛盾）

### 8.6 系统性 / 关联类
- 供应商围标/回扣
- 拼车/同行分别报销
- 审批人-报销人合谋
- 员工消费画像离群
- 节假日前后异常

## 10. 司乘亲密度算法说明

### 10.1 方法 1：集中度指标（先落地）
```
重复率 R = 最高频车牌的行程数 / 该乘客总行程数
命中条件：总行程数 ≥ N  且  R ≥ 阈值
```

### 10.2 方法 2：HHI 赫芬达尔指数
```
HHI = Σ (每个车牌行程占比)²
HHI 越接近 1 越集中（异常）
```

### 10.3 方法 3：概率显著性检验（降误报）
```
期望重复次数 E ≈ n·(n-1) / (2M)
n = 乘客行程数，M = 城市活跃司机数
实际重复次数远大于 E（如 >10 倍 或 P<0.001）判定异常
```

### 10.4 方法 4：图/网络分析（识别团伙）
- 构建"员工—车牌"二部图，边权为搭乘次数。
- 强边 = 一对一绑定；共享车牌子图 = 疑似团伙。

### 10.5 防误报要点
- 设置最小样本量门槛。
- 按城市司机密度动态调整期望值。
- 通勤班车、长期驻场用车加入白名单。
